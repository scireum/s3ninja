S3 ninja

S3 ninja emulates the S3 API for development and testing purposes.

Download Contribute
Screenshot
Install
  1. Download the release as ZIP file.
  2. Unzip into your favorite directory.
  3. To change the configuration create a file instance.conf in the root directory and adapt the settings. Most of the configuration can be found in app/application.conf - Do not modify this file, as it will be overridden with the next patch.
  4. Start the ninja using sirius.sh start (on any UNIX platform with java in the PATH) or java IPL to start it manually
  5. Navigate to http://localhost:9444 - or the port you set with http.port in the config.
  6. Run S3 API-Calls against http://localhost:9444/s3 (e.g. http://localhost:9444/s3/test-bucket/test-object)
  7. Use sirius.sh stop or java -Dkill=true -Dport=9191 IPL to terminate the server
API
Currently the ninja supports only object methods: GET, PUT, HEAD, DELETE. ACLs are also ignored but for non-public buckets, the given Authorization hash is verified using the credentials shown on the page.
